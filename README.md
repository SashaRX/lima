# LIMA — сайт магазина (реализация дизайна)

Самостоятельная реализация дизайна из claude.ai/design (проект «Lima — сайт для магазина»)
в виде **обычного сайта без зависимостей**: один `index.html` + папки с картинками. Никакого
React, сборки или рантайма claude.ai/design — открывается в любом браузере и хостится как статика.

## Как открыть
Просто дважды кликните `index.html` (данные встроены в страницу, поэтому работает и с `file://`).
Шрифты Cormorant/Manrope подгружаются с Google Fonts — нужен интернет для точного вида (иначе
подставятся системные serif/sans).

## Что реализовано (1:1 с дизайном)
- Тёмная тема «Ателье», акцент «Медовый дуб», плотная сетка — как в дизайне по умолчанию.
- Все экраны: **Каталог** (8 товаров), **Карточка товара**, **Корзина**, **О мастерской**,
  **Индивидуальный заказ**, **Доставка и оплата**, **Контакты**, футер.
- Переключение **RU / EN**, фильтр по категориям, **корзина** с количеством и итогом
  (сохраняется в `localStorage`).
- Статичный логотип в липкой шапке (эмблема — встроенный SVG, чёткий в любом размере).
- Мобильный-first адаптив на CSS (media-queries): 4→3→2 колонки, бургер-меню,
  без тяжёлого JS и пере-рендера при ресайзе. Картинки-заглушки ~5 КБ (всего ~56 КБ).

## ⚠️ Картинки товаров — это заглушки
Реальные фотографии в проекте claude.ai/design весят больше лимита передачи через MCP
(256 КБ на файл), поэтому сюда они **не выгрузились**. Вместо них — аккуратные
деревянно-тонированные градиенты-заглушки на тех же путях. Сайт из-за этого выглядит цельно,
но фото — ненастоящие.

**Чтобы поставить реальные фото:** скачайте оригиналы из проекта claude.ai/design (или возьмите
свои) и положите их с этими именами, заменив заглушки:

```
assets/handles-dark.png      ← фон hero и фото на «О мастерской»
assets/board-bread.png       ← фото на «Контакты»
catalog/boards/board-bread.png         ← Доска «Торцевая»
catalog/tableware/sushi-white.png      ← Набор «Дзен»
catalog/trays/sushi-table.png          ← Сервировка «Венге»
catalog/accessories/paper-holder.png   ← Держатель полотенец
catalog/tableware/set-box.png          ← Подарочный набор
catalog/accessories/rolling-pin.png    ← Скалка из бука
catalog/accessories/handles-dark.png   ← Рукояти «Венге»
catalog/decor/chair.png                ← Кресло «Lima»
```

Имена и пути менять не нужно — просто перезапишите файлы. Размер любой (CSS впишет по месту).
`_tools/gen-placeholders.mjs` — генератор заглушек (если нужно пересоздать): `node _tools/gen-placeholders.mjs`.

## Админка (`admin.html`)
Каталог теперь хранится в **`products.json`**, и его можно править через панель администратора.

- **Вход:** открой `https://sasharx.github.io/lima/#admin` (перекинет на `admin.html`), введи пароль.
  Пароль по умолчанию `lima2025` — поменяй его в `admin.html` (строка `var ADMIN_PASS = "..."`).
  Важно: пароль на статическом сайте лишь прячет панель; реальная защита публикации — это токен GitHub.
- **Что умеет:** добавлять/редактировать/удалять карточки (RU+EN: название, материал, описание,
  характеристики, цена, категория), загружать фото товаров.
- **Публикация в 1 клик:** в «Настройки публикации» вставь персональный токен GitHub
  (github.com/settings/tokens — fine-grained на репозиторий `lima`, Contents: Read and write;
  или classic со scope `repo`). Кнопка «Опубликовать» коммитит `products.json` и новые фото
  (в `catalog/uploads/`) прямо в репозиторий `lima` → сайт сам пересобирается за ~1 мин.
  Токен хранится только в твоём браузере (localStorage), на сервер проекта не уходит.

**`deploy.ps1` и админка не конфликтуют:** деплой публикует КОД (`index.html`, `admin.html`,
стили, картинки-заглушки) и **сохраняет** `products.json` и `catalog/uploads/`, созданные в админке.

## Данные / запасной вариант
Каталог — в `products.json` (правится админкой). Та же копия встроена в `index.html` и `admin.html`
как запасной вариант, чтобы всё работало и офлайн (`file://`), когда `products.json` недоступен.
Тексты интерфейса (RU/EN) — в `I18N` внутри `index.html`. Телефон/почта-заглушки:
`+7 900 000-00-00`, `hello@lima-wood.ru`, `@lima.wood`.
